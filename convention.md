
# Git Commit Convention

## Format

():

## Types
- feat: new feature
- fix: bug fix
- refactor: code refactor
- docs: documentation
- test: tests
- chore: build / config

## Scopes
- api
- batch
- core
- build
- docs

## Rules
- use imperative, present tense
- no period at the end
- keep subject under 50 chars

## Examples

- feat(api): add migration GUI page
- feat(batch): add user migration job
- refactor(core): extract json mapper
- chore(build): setup multi-module gradle
- fix(api): resolve thymeleaf template error
- docs: add commit convention
